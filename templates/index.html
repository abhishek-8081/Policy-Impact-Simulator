About

Policy Impact Simulator is an integrated modelling workbench for exploring how energy policy decisions interact with macroeconomic outcomes.

CLEWS Energy Model
Computes energy cost, total emissions, renewable investment requirements, and consumption based on carbon tax, renewable share, and baseline cost parameters. Grounded in the CLEWS/OSeMOSYS framework used in national energy planning.
OG-Core Macro Model
Runs an overlapping-generations (OLG) macroeconomic model to derive GDP, wages, household savings, capital stock, and government revenue. Captures how fiscal policy and energy costs affect long-run economic equilibrium.
Coupled Pipeline
Chains both models together with a validated mapping layer. Run CLEWS first and feed energy cost outputs into OG-Core as production cost shocks — or reverse the direction to study how GDP growth changes energy demand.
Converging Equilibrium
Iterates the coupled pipeline until the absolute change in GDP between rounds falls below a user-set tolerance. Finds the joint equilibrium of the energy–economy system and visualises the convergence path across iterations.
Scenario Management
Create, save, and switch between named scenarios. Country presets (India, Germany, Nigeria, USA) provide real-world baseline parameters. Every run is logged with full timing and outputs for later retrieval and comparison.
Analysis Tools
Sensitivity analysis identifies which parameter drives each output most strongly. Parameter sweep charts how any single input affects all outputs across its full range. Side-by-side run comparison shows absolute and percentage differences between scenarios.
How to Use
1
Create a Scenario
On the Models tab, type a name in the Scenario field and click + New Scenario. A scenario stores your parameter values and run config so you can reproduce results later.
2
Set Parameters
Use the sliders or number inputs to adjust Carbon Tax, Base Energy Cost, Renewable Share, Tax Rate, and Population Growth. Or click a Country Preset to load real-world baseline values instantly. Hover the i icons for explanations of each parameter.
3
Choose a Model and Run
Select CLEWS (energy/climate) or OG-Core (macro/fiscal) from the Model panel, then click Run. Result cards appear with the computed outputs. A + / - indicator shows how each value changed from the previous run.
4
Run the Coupled Pipeline
Switch to the Coupled tab. Choose a direction (CLEWS to OG-Core, or reverse), then click Run Coupled Pipeline. The Data Flow diagram shows each stage, timing, and the intermediate exchange values passed between models.
5
Find Equilibrium with Converging Mode
Go to the Converging tab. Set a Tolerance (e.g. 0.5) and Max Iterations, then click Run Converging. The pipeline iterates until the absolute GDP change per iteration falls below the tolerance. Three charts show GDP trajectory, emissions, and the convergence delta.
6
Compare and Export
On the Results tab, browse all saved runs. Select any two runs in the Compare dropdowns to see a side-by-side bar chart and a table of % differences. Click CSV on any run to download its full output.
FAQ
What is the Policy Impact Simulator?
A prototype tool that demonstrates bidirectional integration between CLEWS/OSeMOSYS (energy system model) and OG-Core (overlapping-generations macroeconomic model). It lets analysts explore how energy policies ripple through the broader economy and vice versa.
What is CLEWS / OSeMOSYS?
CLEWS (Climate, Land, Energy and Water Systems) is a framework for integrated resource planning. OSeMOSYS is an open-source energy system optimisation model widely used in developing-country long-range planning. This simulator uses a lightweight analytical approximation of the energy cost and emissions outputs.
What is OG-Core?
OG-Core is an open-source overlapping-generations (OLG) macroeconomic model maintained by the Policy Simulation Library. It models how households in different age cohorts save, work, and consume over their lifetimes. This simulator uses a Cobb-Douglas approximation of its GDP, wages, and capital outputs.
What does "Coupled" mean?
In Coupled mode the two models run sequentially and exchange data once. Model A runs first, a mapping layer transforms its outputs into inputs for Model B, and Model B runs second. You can choose either direction: CLEWS drives OG-Core (energy costs affect the macro economy) or OG-Core drives CLEWS (GDP growth feeds back into energy demand).
When has the Converging pipeline "converged"?
The pipeline runs CLEWS and OG-Core in a loop, feeding each iteration's macro output back into the energy model. It is considered converged when the absolute change in GDP between two consecutive iterations (|ΔGDP|) falls below the Tolerance value you set. A green star marks the convergence point on the GDP chart.
What are the country presets based on?
The four presets (India, Germany, Nigeria, USA) use indicative values drawn from public sources: IEA energy statistics, World Bank carbon pricing databases, and OECD fiscal data. They are approximate baselines suitable for demonstrating relative differences between economies, not precise policy simulations.
What does Sensitivity Analysis show?
Sensitivity Analysis perturbs each input parameter by 10% and measures how much each output changes. The result is an elasticity: % change in output per 1% change in input. The table highlights which single parameter has the largest influence on each output metric at the current parameter values.
Why does a run show a yellow warning badge?
Warnings are non-blocking alerts for parameter values that are technically valid but economically unusual — for example a carbon tax above $150/tCO₂, a renewable share above 85%, or a tax rate above 50%. The run still executes; the warning is there to prompt you to double-check your intent.
How do I export results?
On the Results tab, each run row has a CSV button that downloads all numeric outputs for that run as a flat key-value CSV. A JSON export is also available via the API at /results/<run_id>/export/json. Individual scenarios can be exported via /scenarios/<id>/export.
Are these real model outputs?
No — this prototype uses analytical approximations (closed-form formulas) to stand in for the full OG-Core optimisation solver and OSeMOSYS LP engine. The qualitative relationships are correct (higher carbon tax raises energy cost, higher tax rate reduces GDP, etc.) but the absolute numbers are illustrative only.

Key Features

Design
Minimalistic Interface
Clean, uncluttered layout built for analysts and policymakers. Every control is labelled, every output is explained, and nothing competes for attention. No installation, no command line — runs entirely in the browser.
Usability
No-Code Policy Exploration
Adjust carbon taxes, renewable share, fiscal rates, and population growth with sliders. Instant feedback with directional indicators showing how each lever shifts outputs. Country presets load real-world baselines in one click.
Modelling
Iterative Convergence Engine
The converging pipeline runs CLEWS and OG-Core in a feedback loop — each round's macro output feeds back into the energy model — until GDP stabilises within a user-set tolerance. Tracks every iteration so you can inspect the convergence path.
Integration
Bidirectional Model Coupling
Run energy-to-economy (CLEWS drives OG-Core) or economy-to-energy (OG-Core drives CLEWS). A validated mapping layer handles unit conversion and data exchange between models, with live pipeline step indicators.
Analysis
Sensitivity & Sweep Analysis
Sensitivity analysis perturbs every parameter by 10% to rank which lever matters most for each output. Parameter sweep sweeps any single input across its full range and charts all outputs simultaneously — without writing a single line of code.
Reproducibility
Scenario Logging & Export
Every run is automatically saved with its full parameter set, timing data, and outputs. Name and save scenarios for later. Compare any two runs side-by-side with a grouped bar chart and a % difference table. Export to CSV in one click.
Validation
Real-time Parameter Warnings
The system checks for economically unusual inputs — extremely high carbon taxes, near-total renewable shares, confiscatory tax rates — and surfaces non-blocking warnings before you run. Runs always complete; warnings are advisory, not blockers.
Accessibility
Light & Dark Mode
Toggle between light and dark themes. All charts, labels, and legends re-render immediately to match the selected theme. Preferences persist across tab switches and remain consistent across all four modules.
